# Hospital-Management-System
This project focuses on designing and building a complete software developemt system for a Fertility Treatment Hospital (IVF).

The system manages patient treatment, medical records, fertility cycles, donors and embryos.

Users:

1. Doctor 
-> View assigned patients
-> Add / update Medical Records
-> Create Appointments
-> Request Test Reports
-> View Embryo status
-> View donor information

2. Hospital Administrator
-> Register patients
-> Assign rooms
-> Manage doctors
-> Manage services
-> Generate patient bills

3. Patient
-> Book appointments
-> View test results
-> View medical record summary
-> See bills
-> Track IVF cycle progress

4. Lab Technician
-> Upload test reports
-> Record embyo development
-> Update donation bank stock
-> Manage lab equipment

User Stories:

Patient (Roberto, Albi)
As a patient, I want to book an appointment with a doctor so that I can receive treatment.
As a patient, I want to view my test results so that I can understand my health status.
As a patient, I want to see my medical bills so that I know how much I must pay.
As a patient, I want to view my medical history.


Doctor (Drini, Christian)
As a doctor, I want to view my patients’ medical records so that I can make accurate diagnoses.
As a doctor, I want to request lab tests so that patient conditions can be analyzed.
As a doctor, I want to track embryo development during IVF treatment.

Administrator (Armend)
As an administrator, I want to register new patients so that they can receive treatment.
As an administrator, I want to assign hospital rooms to patients.
As an administrator, I want to generate patient bills for services used.

Lab Technician(Silvja, Arbri)
As a lab technician, I want to upload test results so doctors can review them.
As a lab technician, I want to update embryo development stages.
As a lab technician, I want to track donation bank inventory



The most important features: 

-> Patient registration and medical records
-> Appointment management
-> IVF cycle and embryo tracking
-> Test report management
-> Donor and donation bank management
-> Patient billing system

The technologies we will be using: 

* Database: MySQL / PostgreSQL
* Design: Figma
* Repository: GitHub
